=========================================
 D/s Collar — Outfits Plugin Setup
=========================================

The Outfits menu reads from your #RLV shared folder. The folder
layout the collar expects is:

    #RLV/
        ~outfits/
            ~base/       ← non-strippable base kit (REQUIRED)
            Casual/      ← one folder per outfit
            Formal/
            BDSM/
            ...

Two things must exist before the Outfits menu will work usefully:

  1. #RLV/~outfits/               — the outfits library root
  2. #RLV/~outfits/~base/         — the protected base kit subfolder

The tilde-prefixed naming is the "orthodox RLV" convention (also
used by OpenCollar). It keeps these folders out of the way at the
bottom of the inventory tree while still being visible to RLV
enumeration commands — which is what lets the collar's Strip menu
hide locked items from the picker before you ever click them.

-----------------------------------------------------------------
 If you are upgrading from an older collar build
-----------------------------------------------------------------

Earlier builds used dot-prefixed names (.outfits / .outfits/.base).
Those folders are no longer recognised — rename them:

    .outfits          →  ~outfits
    .outfits/.base    →  ~outfits/~base

Right-click each folder in your inventory → Rename. Capitalisation
and the exact spelling matter; the leading tilde (~) is required.

-----------------------------------------------------------------
 1. Make sure you have a #RLV shared folder
-----------------------------------------------------------------

In Firestorm (or any RLV-capable viewer):

  Inventory window  →  + (New)  →  New Folder
  Name the folder exactly: #RLV
  (The # symbol is required.)

If a #RLV folder already exists, skip this step.

-----------------------------------------------------------------
 2. Create #RLV/~outfits
-----------------------------------------------------------------

Inside #RLV, create a subfolder named exactly:

    ~outfits

(Yes, the leading tilde is required. It sorts the folder to the
bottom of the inventory tree and signals "system folder, don't
touch" to other RLV-aware tools.)

-----------------------------------------------------------------
 3. Create #RLV/~outfits/~base
-----------------------------------------------------------------

Inside #RLV/~outfits, create a subfolder named exactly:

    ~base

Drop links to your "always on" items into ~base:

  - Your body / mesh head
  - AO / animation overrider
  - The collar itself (optional)
  - Any jewelry, piercings, or implants you treat as permanent
  - HUDs you always wear

How to add a link: right-click an item in your inventory →
"Copy", then in the ~base folder → "Paste as Link". Always link;
never move the original. RLV needs the link, not the item.

The collar automatically locks #RLV/~outfits/~base with
@detachallthis the moment the Outfits plugin is enabled, so
nothing under ~base can be removed by the Strip menu, the Wear
action, the Remove action, or any external relay — even if you
forget to lock items individually.

-----------------------------------------------------------------
 4. Create one folder per outfit under #RLV/~outfits
-----------------------------------------------------------------

Alongside ~base, create one subfolder per outfit. Each holds
links to the items in that look. Example tree:

    #RLV/
        ~outfits/
            ~base/
                (body, AO, collar, jewelry — links)
            Casual/
                (jeans, t-shirt, sneakers — links)
            Formal/
                (dress, heels, gloves — links)
            BDSM/
                (corset, cuffs, gag — links)

The Outfits picker lists everything under ~outfits *except*
folders whose names start with "." or "~", so ~base is hidden
from the picker even though it's protecting your kit.

-----------------------------------------------------------------
 5. Using the Outfits menu
-----------------------------------------------------------------

Tap an outfit in the picker to open its action menu. Five actions
are available; which ones you see depends on the touching avatar's
ACL level.

  Add     — Attach this folder additively. Items already worn at
            slots the new folder would fill are NOT kicked off —
            the new items layer on top. Uses @attachallover.

  Wear    — Replace. Strips worn unlocked items (attachments AND
            system clothing layers) in three phases:
              1. @detachallthis:~outfits=force — clears ~outfits
                 subtree
              2. @remattach=force — attachments worn from outside
                 ~outfits (regular inventory, unlinked HUDs, etc.)
              3. @remoutfit=force — system clothing layers worn
                 from outside ~outfits
            Then attaches the chosen folder. Items under
            #RLV/~outfits/~base are protected and survive every
            phase, as does the collar itself and anything else
            with an explicit lock.

  Remove  — Detach this outfit's items. Blocked silently if the
            outfit is locked.

  Lock    — Apply a folder lock (@detachallthis) to this outfit.
            While locked, the outfit cannot be removed by the
            Strip menu, the Remove action, or external relays.
            Available to ACL 3 (trustee), ACL 4 (self-owned wearer),
            and ACL 5 (primary owner) only.

  Unlock  — Release the lock. Same ACL gating as Lock.

Lock state is **persistent**: it lives in the collar's linkset data
and is automatically re-applied on every script start. Detaching
and re-attaching the collar does NOT clear locks — RLV restrictions
survive the rez cycle because the collar itself is the issuing
object.

A lock is released only when:
  - You tap **Unlock** in the picker for that outfit.
  - You **factory-reset** the collar (Maintenance → Reset). The
    plugin releases viewer-side restrictions explicitly before the
    reset so nothing is left orphaned.

Locked outfits are marked with a "*" in the picker.

-----------------------------------------------------------------
 ACL summary
-----------------------------------------------------------------

  ACL 1 (public)             : Add, Wear, Remove
  ACL 2 (owned wearer)       : Add, Wear, Remove
  ACL 3 (trustee)            : Add, Wear, Remove, Lock, Unlock
  ACL 4 (self-owned wearer)  : Add, Wear, Remove, Lock, Unlock
  ACL 5 (primary owner)      : Add, Wear, Remove, Lock, Unlock

-----------------------------------------------------------------
 Troubleshooting
-----------------------------------------------------------------

"No outfits found in #RLV/~outfits."
    Either ~outfits doesn't exist yet, or it exists but holds no
    outfit subfolders (only ~base counts as protected, not as an
    outfit). Create at least one regular subfolder (e.g. Casual)
    and put item links inside it.

"RLV not responding."
    Make sure RLV (or RLVa) is enabled in your viewer
    preferences. Firestorm: Preferences → Privacy → General →
    "Enable Restrained Love API".

Outfit still won't change.
    Check capitalization on folder names — RLV is case-sensitive.
    Make sure ~base is spelled exactly "~base" (with the leading
    tilde) and lives directly under #RLV/~outfits, not somewhere
    else.

Remove does nothing.
    The outfit is probably locked (look for "*" in the picker)
    or one of its items has its own @detachallthis lock from a
    different source (plugin_folders, an RLV relay, etc.).
    Unlock the outfit first, then Remove.

Strip menu still shows an item I expect to be locked.
    The collar pre-filters the Strip picker against every active
    @detachallthis folder lock it can see. Foreign locks that
    target dot-prefixed folders (legacy ".outfits/.base" or
    private folders from other plugins) are invisible to the
    pre-filter — those items show up once, fail silently when
    clicked, and are hidden for the rest of the session. Tilde-
    prefixed locks (including the collar's own ~outfits/~base)
    are pre-filtered correctly and never appear.
